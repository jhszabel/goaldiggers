2. Fejlesztési terv: mérföldkövek, eredmények

  • 1. fázis (1. hét): Funkcionális követelmények és tervezés
  • 2. fázis (2–3. hét): Adatmodell és architektúra elkészítése
  • 3. fázis (4–5. hét): Backend implementáció (felhasználók, tippek, ranglista)
  • 4. fázis (6–7. hét): Frontend implementáció, felhasználói élmény kialakítása
  • 5. fázis (8. hét): Tesztelés, dokumentáció, bemutató előkészítése

3. Programozói nyelvek, fejlesztő eszközök

  • Backend: Python (Django REST Framework)
  • Frontend: React + JavaScript
  • Adatbázis: PostgreSQL
  • Fejlesztő eszközök: Visual Studio Code, GitHub, Docker

4. Használni tervezett eszközök, technikák

  • Verziókezelés: Git, GitHub
  • CI/CD: GitHub Actions (automatikus build, teszt)
  • Adatkezelés: REST API, JSON formátum
  • Tesztelés: Unit tesztek, funkcionális tesztek
  • Projektmenedzsment: Trello, Teams

5. Leendő produktumok és létrejöttük ideje

  • Követelmény specifikáció – 1. hét vége
  • Rendszerterv – 3. hét vége
  • Alfa verzió (backend) – 5. hét vége
  • Béta verzió (frontend + backend) – 7. hét vége
  • Tesztelt, demózható rendszer + dokumentáció – 8. hét vége

6. Tesztelési követelmények

  • Belépés, regisztráció és adatbiztonság ellenőrzése
  • Tippelési logika (pontozás, időkorlát) helyes működése
  • Ranglista és statisztikák pontossága
  • Felhasználói felület kompatibilitása különböző böngészőkön és eszközökön
  • Stresszteszt minimális terheléssel (pl. több felhasználó egyidejű aktivitása)
